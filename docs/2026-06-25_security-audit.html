Overall Posture

Posture: Adequate — Low Attack Surface.

Egg & Out is a marketing-only website: no logins, no database, no payments taken on-site, no analytics or tracking. That makes the attack surface very small and the practical risk low. The one item worth doing before launch is adding a standard set of security response headers (the site currently ships without them), plus a routine clean-up of two moderate dependency advisories.

Before Launch

Open Items

  • ⚠
    Add security headers  ·  CSP, X-Frame-Options/frame-ancestors, X-Content-Type-Options, Referrer-Policy, Permissions-Policy — via next.config or vercel.json. (Medium · the main item.)
  • ⓘ
    Clear 2 moderate dep advisories  ·  postcss via next; bump to a safe version (avoid a forced major) and re-run npm audit. (Low.)
  • ⓘ
    Confirm the catering email  ·  destination is a placeholder. If the form is ever moved off mailto to a server endpoint, add a honeypot + rate-limit + verified delivery. (Low.)

    Branded 404 / 500 pages  ·  add as part of P5 polish. (Low.)
Recommended

Next Steps

Ship the header suite

One config change covers CSP, clickjacking, MIME-sniffing, referrer and permissions — the single highest-value security task here.

Re-audit at launch

Once the ordering channel and any embeds/backends are chosen, re-check the CSP allow-list, third-party key exposure, and form delivery.

Keep deps current

Resolve the postcss/next advisories at a safe version and make npm audit part of the pre-deploy routine.
